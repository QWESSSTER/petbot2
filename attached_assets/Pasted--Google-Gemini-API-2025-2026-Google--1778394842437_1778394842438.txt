Да, это типичная проблема Google Gemini API в 2025–2026.

Причина не в твоём коде.
У Google сейчас часто бывает:

* free tier = 0
* региональные ограничения
* ключ создан, но quota не активирована
* проект автоматически уходит в restricted mode

Особенно часто это происходит:

* на РФ/IP из СНГ
* на новых аккаунтах
* на подозрительных VPN
* на recycled Google accounts

⸻

Почему у тебя 404

Это не «модель не существует».

Gemini API специально возвращает:

* 404
* PERMISSION_DENIED
* quota=0

когда:

* у проекта нет доступа к модели
* free quota отключена
* API не активирован полностью

⸻

Новый extract_from_image

async def extract_from_image(image_data: bytes) -> dict:
    base64_image = base64.b64encode(image_data).decode("utf-8")
    response = client.chat.completions.create(
        model="qwen/qwen2.5-vl-72b-instruct:free",
        messages=[
            {
                "role": "user",
                "content": [
                    {
                        "type": "text",
                        "text": """
Извлеки информацию о заведении.
Верни JSON:
{
  "name": "",
  "address": "",
  "hours": "",
  "avg_price": "",
  "promotions": ""
}
Если поля нет — null.
"""
                    },
                    {
                        "type": "image_url",
                        "image_url": {
                            "url": f"data:image/jpeg;base64,{base64_image}"
                        }
                    }
                ]
            }
        ]
    )
    text = response.choices[0].message.content
    try:
        return json.loads(text)
    except:
        return {
            "name": None,
            "address": None,
            "hours": None,
            "avg_price": None,
            "promotions": None
        }
